## 一、專題概述

### 1.1 專題名稱
**TripFlow** — AI 驅動的智慧旅遊規劃平台

### 1.2 專題簡介
TripFlow 是一個結合 **AI 行程規劃** 與 **社群分享** 的旅遊網站。使用者可以在互動式地圖上搜尋景點、透過 AI 自動生成多天旅遊路線，並將規劃好的行程分享到社群牆供其他旅人參考。

### 1.3 核心功能
| 功能模組 | 說明 |
|---------|------|
| 互動地圖規劃 | 在 Google Maps 上搜尋景點、建立多日行程、規劃路線 |
| AI 智慧規劃助手 | 輸入自然語言需求，由 Gemini AI 自動生成完整行程 |
| 行程儲存管理 | 登入後可儲存、載入、刪除個人行程 |
| 靈感社群（社群牆）| 將行程發佈至社群，支援按讚、留言、瀏覽 |
| 熱門榜單 | 統計所有用戶行程，排出最多人造訪的景點 |
| 會員系統 | 註冊、登入、個人資料管理、帳號刪除 |

### 4.2 關聯式綱要圖（Relational Schema）

```
users(id, name, email, password, email_verified_at, remember_token, created_at, updated_at)
  PK: id

trips(id, user_id, title, itinerary_data, chat_history, created_at, updated_at)
  PK: id
  FK: user_id → users(id)  [CASCADE DELETE]

posts(id, user_id, trip_id, title, content, image_url, days_count, views_count, created_at, updated_at)
  PK: id
  FK: user_id → users(id)  [CASCADE DELETE]
  FK: trip_id → trips(id)  [SET NULL on DELETE]

comments(id, user_id, post_id, content, created_at, updated_at)
  PK: id
  FK: user_id → users(id)  [CASCADE DELETE]
  FK: post_id → posts(id)  [CASCADE DELETE]

post_likes(id, user_id, post_id, created_at, updated_at)
  PK: id
  FK: user_id → users(id)  [CASCADE DELETE]
  FK: post_id → posts(id)  [CASCADE DELETE]

places(id, title, description, address, latitude, longitude, rating, created_at, updated_at)
  PK: id

place_stats(id, place_id, name, city, type, saved_count, likes_count, views_count, score, created_at, updated_at)
  PK: id
  FK: place_id → places(id)
```

## 五、Model 關聯對應

```
User
 ├── hasMany(Trip)          一個用戶可有多個行程
 ├── hasMany(Post)          一個用戶可發多篇貼文
 ├── hasMany(Comment)       一個用戶可留多則留言
 └── belongsToMany(Post)    透過 post_likes 中介表（多對多）

Trip
 └── belongsTo(User)        每個行程屬於一個用戶

Post
 ├── belongsTo(User)        每篇貼文屬於一個用戶
 ├── belongsTo(Trip)        每篇貼文可關聯一個行程（可空）
 ├── hasMany(Comment)       一篇貼文可有多則留言
 └── hasMany(PostLike)      一篇貼文可有多個按讚記錄

Comment
 ├── belongsTo(User)        每則留言屬於一個用戶
 └── belongsTo(Post)        每則留言屬於一篇貼文

PostLike
 ├── belongsTo(User)
 └── belongsTo(Post)
```

## 八、Services 說明

### `PlaceStatisticsService`
**用途**：從所有用戶的行程資料（itinerary_data JSON）中，統計最多人加入的景點。

**運作方式**：
1. 用 `cursor()` 逐筆讀取所有 Trip，節省記憶體
2. 解析每個 Trip 的 JSON，統計景點出現次數
3. 過濾掉「出發」「回家」「返程」等非景點關鍵字
4. 依出現次數排序，回傳前 N 名
5. 同時帶回景點的 Google 照片與評分

**被使用於**：`LandingController`（取前 6）、`RankingController`（取前 15）

## 十、系統安全機制

| 機制 | 說明 |
|------|------|
| **CSRF 保護** | 所有 POST/DELETE 請求需帶 CSRF Token，防止跨站請求偽造 |
| **Rate Limiting** | AI 生成 API 每用戶每 60 秒限 1 次，防止 API Key 被濫用 |
| **Authorization** | 刪除行程/貼文/留言前，驗證是否為本人所有（`user_id === Auth::id()`）|
| **密碼雜湊** | 使用 bcrypt 演算法（rounds=12）儲存密碼，不存明文 |
| **XSS 防護** | Blade 模板預設 `{{ }}` 自動 HTML Escape，AJAX 留言使用 `escHtml()` 函數 |
| **SQL Injection** | 使用 Eloquent ORM + Query Builder，全程參數化查詢 |
| **檔案上傳驗證** | 限制 MIME 類型（jpeg/png/jpg/gif/webp）及檔案大小（5MB 以內）|
| **Session 保護** | Session 以資料庫儲存，支援 Session ID 輪換 |

## 十二、專題亮點與特色技術

### 12.1 多天行程記憶系統
- AI 對話紀錄（chat_history）以 JSON 存入資料庫，下次載入行程時恢復上下文
- SessionStorage 暫存草稿，登出/切換帳號自動清除

### 12.2 智慧停車資訊
- AI 回傳 `parking_mode: INTERNAL / EXTERNAL`
- 前端自動顯示「附設停車場（直接導航）」或「搜尋附近停車場」連結

### 12.3 替代路線切換
- Google Directions API 回傳複數路線，用彩色 Polyline 繪製
- 點擊替代路線可即時切換，顯示省時/多花幾分鐘標籤

### 12.4 社群照片來源雙軌
- 優先使用用戶手動上傳的圖片（存入 `storage/app/public/posts/`）
- 若無上傳，自動從行程景點的 Google Places 照片擷取第一張作為封面

### 12.5 熱門景點演算
- `PlaceStatisticsService` 掃描全站所有行程 JSON
- 統計景點出現頻率 → 排序 → 首頁顯示前 6，榜單頁顯示前 15

## 十三、快速啟動說明（給需要在本機跑的人）

```bash
# 1. 安裝 PHP 套件
composer install

# 2. 安裝前端套件
npm install

# 3. 複製環境設定
cp .env.example .env
php artisan key:generate

# 4. 在 .env 填入 API Key
GOOGLE_MAPS_API_KEY=你的key
GEMINI_API_KEY=你的key

# 5. 建立資料庫
touch database/database.sqlite
php artisan migrate

# 6. 建立 storage 連結（圖片上傳）
php artisan storage:link

# 7. 啟動前端（另開一個終端機）
npm run dev

# 8. 啟動後端
php artisan serve
# 或使用 Laragon，直接開啟 http://travel_planner.test
```
